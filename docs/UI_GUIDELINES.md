# UI and UX Guidelines

## Design objective

The platform should feel clear, calm and operationally safe for members and LT users. The interface should explain the next action without requiring training.

## Content hierarchy

### Member detail

1. Current total score and light colour
2. Precise improvement actions
3. Category scores
4. Monthly history
5. Recap and downloadable card

### LT member trend

1. Latest performance
2. Change from previous month
3. Change over the recorded period
4. Monthly score cards
5. Seven category comparisons
6. Collapsed detailed table

## Responsive rules

- Normal mobile use must not depend on a wide table.
- Tables are secondary detail views and should be collapsed by default.
- When a wide table is opened, freeze the month and total-score columns.
- Use card layouts that collapse from two columns to one column below 620 px.
- Interactive controls should be at least 44 px high.
- Avoid essential information being communicated by colour alone.

## Friendly language

Use clear action labels:

- Publish as Latest Month
- Add to History
- Replace YYYY-MM Data
- Cancel Upload
- View Full Monthly Data

Avoid technical database language and ambiguous labels such as Submit, Run or Process.

## Feedback states

Every upload action must provide:

- detected filename;
- detected month;
- member count;
- publication mode;
- validation errors;
- non-blocking warnings;
- final success message.

## Accessibility

- Provide visible keyboard focus.
- Maintain readable text contrast.
- Do not use small grey text for critical warnings.
- Keep mobile body text at a comfortable size.
- Ensure touch targets have sufficient spacing.

## Visual consistency

- Wine red is the primary action and brand colour.
- Green, yellow, red and black are reserved for traffic-light meaning.
- Destructive or replacement actions require explicit confirmation.
- Cards should use consistent radius, spacing and soft shadows.
