# BNI 紅黃綠燈計算平台 — Workflow & 說明

> 目標:每月收到 BNI Member Traffic Light Excel 後,一 run 即完成「覆核 → 分析 → 生成個人化 recap」,唔使人手逐個計。
> 分會:Bingo (Chinese) / BNI Zenith 系統。語言:廣東話繁體中文。

---

## 0. 專案結構

```
bni-traffic-light/
├── index.html                 ← Phase 1 主工具(本機開,離線運算,唔上網)
├── vendor_xlsx.full.min.js    ← SheetJS(本機讀 Excel 用,離線)
├── WORKFLOW.md                ← 本文件
├── SCORING.md                 ← 紅黃綠燈計分邏輯(權威版)
├── samples/                   ← 範例檔(真實月報 + 官方計算器 template)
└── data/                      ← (Phase 2 用)匯出俾會員自助版嘅 JSON
```

## 1. 私隱原則(重要)

- **Phase 1 全程喺瀏覽器本機運算,Excel 唔會上傳去任何伺服器/雲。** 成員姓名 + 生意額安全。
- 唔好將含真實成員數據嘅檔案 commit 上任何公開 repo。
- Phase 2 會員自助版一定要用「私隱式登入」方案(Apps Script + Google 登入,每人只睇到自己),唔可以放公開 GitHub Pages。

## 2. 每月操作流程(LT 用)

1. 由 BNI Connect / 系統下載當月 `Member Traffic Light - Bingo (Chinese) (YYYY-MM).xlsx`。
2. 打開 `index.html`(直接 double-click,用 Chrome/Edge)。
3. **上傳當月檔案**(必需);如想覆核,順便**上傳上月檔案**(選填)。
4. 睇【覆核】頁:系統會自動重算每人分數同 Excel 內分數對比,標出:
   - ❌ 分數對唔上(公式/入機可能有錯)
   - ⚠️ 異常值(週數 = 0、新會員週數過少)
   - 🔻 相對上月燈號跌 / 大幅變動
   - **簽核**:逐項打勾「已核實」→ 填覆核人姓名 → 撳「✅ 確認覆核通過」。系統記低批核人 + 時間(存本機 localStorage,按月份分開)。未簽核時,【批量Recap】頁會提醒。
5. 睇【全會 Dashboard】:綠/黃/紅/黑分佈、達標進度(對比可編輯目標)。
6. 睇【會員】頁:揀會員 → 睇燈號、雷達圖、差邊項、**升到綠燈最平路線**、一鍵 copy recap、**⬇️ 下載會員圖卡(PNG)**。
7. 【批量 Recap】:一次生成全部會員 recap,逐個 copy 文字 + ⬇️ 下載圖卡,一齊 send 落 WhatsApp/WeChat 派發。

## 會員圖卡
- 每個會員一張 PNG 圖卡:BNI BINGO 品牌 + 燈號 + 雷達圖 + 7 項分項 + 升燈貼士 + 頁尾聯絡。
- 全程本機 SVG→Canvas→PNG 生成,離線、唔上網。檔名 `{姓名}-{YYYYMM}.png`。
- 用法:copy recap 文字 + 下載圖卡,一齊發俾會員(文字講重點,圖卡一睇就明)。

## 覆核簽核
- 目的:LT 覆核完先派 recap,留低「邊個、幾時」批核記錄。
- 逐項核實(打勾)+ 覆核人 + 確認 → 綠色 banner「已由 XXX 於 YYYY-MM-DD HH:MM 確認覆核通過」。
- 有「分數對唔上」未打勾時,確認會彈提示,由 LT 決定是否照過。
- 記錄存本機瀏覽器,按月份(期間)分開;可「撤銷」重簽。

## 3. 計分邏輯(摘要,詳見 SCORING.md)

五大元素滿分 100;週數 = P+A+L+M+S。

| 元素 | 滿分 | 門檻 |
|---|---|---|
| 培訓 T | 10 | 0→0 / 1→5 / ≥2→10 |
| 缺席 A | 15 | 0→15 / 1→10 / 2→5 / >2→0 |
| 準時 L | 10 | 0→10 / 1→5 / ≥2→0 |
| 1-2-1(/週) | 10 | ≥1→10 / >0.5→5 / ≤0.5→0 |
| 引薦(比出G/週) | 20 | ≥1.5→20 / ≥1.2→15 / ≥1→10 / ≥0.75→5 / <0.75→0 |
| Biz Give($) | 15 | ≥50萬→15 / ≥20萬→10 / ≥10萬→5 / <10萬→0 |
| 嘉賓(V/週) | 20 | ≥0.75→20 / ≥0.5→15 / ≥0.25→10 / ≥0.1→5 / <0.1→0 |

燈號:綠 ≥70、黃 ≥50、紅 ≥30、黑 <30。

## 4. 可編輯設定(admin)

- 綠燈大目標 %(每屆可改)
- as-of 日期、預測月份、綠燈專員/聯絡人名(用喺 recap)
- 綠燈分數線(預設 70,一般唔使改)

## 5. 會員版 — 自含檔案分享(最簡單,唔使打指令)

**做法**:LT 工具一撳生成一個**自含嘅 HTML**(數據內嵌),直接 send 俾會員或放上網。會員開檔揀自己名就睇到燈號/雷達圖/升燈貼士。

**每月流程(LT)**:
1. 開 LT 工具(`index.html` 網址版或單一檔案)→ 上傳月報。
2. 去【① 上傳】頁底「生成會員版(可分享)」。
3. 密碼**選填**:留空 = 會員一開就見(免密碼);填咗 = 會員要入密碼先解到(AES-256 加密)。
4. 撳「⬇️ 生成會員版 HTML」→ 下載 `BINGO紅黃綠燈-YYYYMM.html`。
5. 將個檔 send 落會員 WhatsApp/WeChat 群(或者放上任何網頁空間)。會員開檔揀名睇自己。

**點解咁樣最簡單**:唔使每月 git、唔使上傳 GitHub、唔使後端。一個檔搞掂,離線都開到。

**私隱**:
- 會員版只含分數/燈號/貼士/recap,**唔含 Biz Give 原始生意額**。
- 想加保護就填密碼(加密)。免密碼版任何攞到個檔嘅人都睇到全會,自己衡量。
- `member.html` = 生成用嘅範本(LT 工具會 fetch 佢);`bni-bingo-*.json` / xlsx 已 `.gitignore`,絕不上傳。

> 註:如果將來想要「固定網址、會員自己上網登入」,`member.html` 已支援放一個加密 `member-data.json` 落 Pages 嘅做法(見 git 版本歷史),不過自含檔案分享對非技術 LT 最易用。
